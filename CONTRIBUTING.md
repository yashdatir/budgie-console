# Contributing to Budgie Console

Thank you for your interest in contributing! We welcome all developers to help improve this project.

## Code of Conduct

We are committed to providing a welcoming and inclusive environment. All contributors must:
- Be respectful and professional
- Welcome diverse perspectives
- Report inappropriate behavior to maintainers

## Getting Started

### Fork and Clone
```bash
git clone https://github.com/yourusername/budgie-console.git
cd budgie-console
```

## Finding Issues to Resolve

1. Visit our [Issues](https://github.com/yourusername/budgie-console/issues) page
2. Look for labels: `good first issue`, `help wanted`, `bug`, `enhancement`
3. Comment on an issue to express interest before starting work

## Creating a Branch

```bash
git checkout -b feature/issue-number-description
# Example: feature/123-add-dark-mode
```

## Making Changes

- Write clear, concise commit messages
- Follow the existing code style
- Add tests for new functionality
- Update documentation as needed

## Submitting a Pull Request

```bash
git push origin feature/issue-number-description
```

1. Create a pull request with a descriptive title
2. Reference the issue: `Closes #123`
3. Describe your changes clearly

## We Need Your Help!

Developers of all skill levels are encouraged to tackle open issues. Even small contributions matter. Start with `good first issue` labels to build familiarity with the codebase!

---

**Have questions?** Open a discussion or comment on an issue. We're here to help!
